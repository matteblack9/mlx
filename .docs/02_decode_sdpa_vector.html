1. 시나리오 — 첫 답변 토큰 생성

1번 문서(Prefill) 에서 40토큰 프롬프트를 처리했다. 이제 모델이 답을 시작한다:

💬 프롬프트: "대한민국의 수도와 …설명해줘" (40토큰, KV 캐시에 저장됨)

🤖 모델이 생성할 첫 토큰: "대"  →  그 다음 "한", "민", "국", "의", "수", "도", "는"…

매 스텝마다 쿼리는 오직 1개(직전 토큰). 그 1개가 지금까지의 KV 캐시 전체를 본다.

이 시나리오의 attention 모양 변화

스텝새 쿼리(qL)KV 길이(N)attention 모양
Decode #1 ("대")140 (프롬프트)S = [1 × 40]
Decode #2 ("한")141S = [1 × 41]
Decode #3 ("민")142S = [1 × 42]
… Decode #1001139S = [1 × 139]

모델 설정은 동일 (Llama 3 8B): B=1, Hq=32, Hkv=8, gqa_factor=4, head dim D=128. Decode #1 을 기준(N=40)으로 따라간다.

"128~512 dim": head dim D. vector 커널은 {64, 96, 128, 256} 을 지원(scaled_dot_product_attention.cpp:622). 코드의 qk_per_thread = D / 32 가 lane 당 담당 dim 을 정하므로, D=128→4, D=256→8, D=512면 lane 당 16 — 다만 512 는 미지원이라 fallback. 이 문서는 D=128 (lane 당 4 dim) 로 따라간다.

2. 왜 decode 는 vector(GEMV) 인가?

Decode 의 attention

S = Q[1,128] @ Kᵀ[128,40] = [1,40]
O = softmax(S) @ V[40,128] = [1,128]
  • 쿼리가 1행 → 벡터 × 행렬 = GEMV
  • KV 캐시(40×128×2)를 한 번씩만 읽고 버림
  • 연산량 << 메모리 읽기량 → memory-bound

MMA(prefill 방식)를 쓰면?

  • 8×8 fragment 인데 쿼리가 1개 → 7/8 이 낭비
  • 행렬곱 하드웨어의 이점이 사라짐
  • 오히려 단순 dot product + simd_sum 이 빠름

그래서 decode 는 MMA 없이, 32 lane 협력 dot product 로 처리.

비유: 도서관에서 책 찾기

Prefill = 40명이 동시에 책을 찾음. 정답표를 크게 펼쳐 기계로 대조하는 게 이득(MMA).

Decode = 나 혼자(쿼리 1개) 책을 찾는데, 서가(KV 캐시)가 40칸이다. 일은 적지만 서가 40칸을 다 훑어야 한다 → 병목은 "계산"이 아니라 "서가 왕복(메모리)". 그래서 여러 사람(SIMD group)이 서가를 나눠 훑는 게 핵심.

6. dot product — simd_sum 으로 32 lane 합치기

한 SIMD group(SG0)이 키 0 의 score q·k₀ 를 계산하는 과정. head dim 128 을 32 lane 이 4개씩 나눠 가졌으므로, 각 lane 이 부분합을 만들고 → simd_sum 으로 32 lane 을 한 방에 합친다.

// sdpa_vector.h:110
for (int j = 0; j < qk_per_thread; j++)  k[j] = keys[j];   // 내 dim 4개 load
U score = 0;
for (int j = 0; j < qk_per_thread; j++)  score += q[j]*k[j]; // 부분합 (내 4 dim)
score = simd_sum(score);                  // ★ 32 lane 부분합 → 완전한 dot product

7. 스레드별 online softmax (루프 본체)

각 SIMD group 은 자기가 맡은 키들을 보며 prefill 과 같은 online softmax 를 돌린다. 단, 여기선 SIMD group 별로 독립적인 max·sum·출력 누적기를 갖는다 (sdpa_vector.h:124):

U new_max = max(max_score, score);
U factor    = fast::exp(max_score - new_max);   // 이전 누적 재보정
U exp_score = fast::exp(score - new_max);       // 이번 score 의 가중치
max_score = new_max;
sum_exp_score = sum_exp_score*factor + exp_score;
for (int j = 0; j < v_per_thread; j++)          // 출력도 같이 누적
  o[j] = o[j]*factor + exp_score*values[j];

루프가 끝나면 각 SIMD group 은 "자기가 본 키들에 대한" 부분 출력 o[4] 와 부분 max/sum 을 갖는다. SG0 은 키 {0,32}, SG1 은 {1,33}, … 의 부분 결과. 이걸 32개 합쳐야 진짜 답이 된다 → 8장.

루프 종료 시점 (N=40, SG 32개 각자의 부분 결과) SG0: 키{0,32} 본 결과 → max₀, sum₀, o₀[0..3] SG1: 키{1,33} 본 결과 → max₁, sum₁, o₁[0..3] SG2: 키{2,34} 본 결과 → max₂, sum₂, o₂[0..3] ⋮ SG31: 키{31} 본 결과 → max₃₁,sum₃₁,o₃₁[0..3] → 이 32개를 "하나의 정확한 softmax 가중평균" 으로 합쳐야 함 (다음 장)

8. SIMD group 32개 결과 합치기 — transpose 트릭

문제: 32개 SIMD group 의 부분 결과를 합쳐야 하는데, SIMD group 끼리는 register 를 직접 못 본다(서로 다른 warp). 해결: threadgroup memory(SRAM)를 경유.

8a. max·sum 부터 합치기 sdpa_vector.h:152

if (simd_lid == 0) {                       // 각 SG 의 대표 lane(0번)이
  max_scores[simd_gid]     = max_score;    // SRAM 에 자기 SG 의 max 기록
  sum_exp_scores[simd_gid] = sum_exp_score;
}
threadgroup_barrier(mem_flags::mem_threadgroup);   // 32개 다 쓸 때까지 대기

max_score = max_scores[simd_lid];          // 이제 lane i 가 SG i 의 max 를 읽음
U new_max = simd_max(max_score);           // 32개 max 중 진짜 최댓값 (1 명령)
U factor  = fast::exp(max_score - new_max);
sum_exp_score = simd_sum(sum_exp_scores[simd_lid]*factor);  // 32개 sum 보정합
역할 전환에 주목: 방금 전까지 simd_lid(lane)는 "head dim 담당"이었는데, 여기선 lane i 가 "SIMD group i" 를 대표해 32개 SG 값을 읽는다. 32개를 32 lane 에 한 줄로 늘어놓고 simd_max/simd_sum 한 방에 합치는 것.

8b. 출력 합치기 — transpose 트릭 sdpa_vector.h:163

출력 o[4] 는 각 SG 가 "같은 dim"(lane 매핑 동일)에 대해 부분합을 갖고 있다. 32개 SG 의 같은 dim 값을 더해야 한다. 그런데 dim 은 lane 이, SG 는 warp 이 들고 있어 축이 엇갈린다 → SRAM 에 썼다가 전치해서 읽어 축을 맞춘다.

for (int i = 0; i < v_per_thread; i++) {
  outputs[simd_lid*BD + simd_gid] = o[i];          // (1) 쓰기: [lane][sg] 위치에
  threadgroup_barrier(mem_flags::mem_threadgroup);
  o[i] = simd_sum(outputs[simd_gid*BD + simd_lid]*factor); // (2) 전치 읽기 + 32 SG 합산
  o[i] = sum_exp_score == 0 ? o[i] : (o[i]/sum_exp_score); // (3) 정규화
  threadgroup_barrier(mem_flags::mem_threadgroup);
}
transpose 트릭 — 왜 인덱스를 바꿔 읽나 쓰기: outputs[ simd_lid*32 + simd_gid ] = o[i] 행 = lane(=dim), 열 = SG ← SG 별 부분합이 "열" 방향으로 늘어섬 ┌────────────── 열 = SG 0..31 ──────────────┐ │ row(lane0): [SG0][SG1][SG2] ... [SG31] │ ← 같은 dim 의 32개 SG 부분합 │ row(lane1): [SG0][SG1] ... │ │ ⋮ │ └───────────────────────────────────────────┘ 읽기: outputs[ simd_gid*32 + simd_lid ] ← 행/열을 바꿔 읽음(전치) → 이제 한 lane 이 "한 dim 의 32 SG 값"을 32 lane 에 펼쳐 읽게 됨 → simd_sum 으로 32 SG 합 = 그 dim 의 최종 분자 핵심: "SG 간 합산"을 SRAM 전치로 "lane 간 합산(simd_sum)"으로 바꾼 것.

9. 긴 컨텍스트용 2-pass 변형

컨텍스트가 아주 길어지면(예: N≥1024, 또는 GQA + N≥4096) 단일 패스는 비효율적이다 — 32 SIMD group 으로 수천 토큰을 훑으면 각 SG 가 너무 많은 일을 직렬로 한다. 그래서 K 를 더 잘게 분할(blocks)해 2단계로 처리한다 (sdpa_vector.h:180).

Pass 1 — sdpa_vector_2pass_1

  • grid 에 block 축(z) 추가 → 여러 block 이 KV 를 병렬 분담
  • 각 block 이 자기 구간의 부분 O, sum, max 를 계산
  • 중간 결과를 임시 버퍼(partials, sums, maxs)에 기록

Pass 2 — sdpa_vector_2pass_2

  • 각 block 의 부분 결과를 읽어 합산
  • block 들의 max 로 전체 max → factor 보정 → 최종 정규화
  • 8b 와 같은 transpose 트릭으로 dim 별 합산 후 DRAM 기록
// Pass 1: block_idx 축으로 KV 를 병렬 분담 (sdpa_vector.h:263)
for (int i = block_idx; i < N; i += blocks) { ... }   // block 마다 KV 의 1/blocks 만
// → sums[o_offset*blocks + block_idx], maxs[...], out[... block_idx ...] 에 부분결과

// Pass 2: blocks 개 부분결과를 online-softmax 로 합침 (sdpa_vector.h:354)
for (int b = 0; b < blocks/BN; ++b) max_score = max(max_score, maxs[...]);
max_score = simd_max(max_score);                       // 전체 max
... factor = exp(maxs[b]-max_score); sum += factor*sums[b]; ...  // 합산
o[i] += factor * partials[i];                          // 부분 출력 합산
핵심 아이디어는 동일: online softmax 는 결합법칙이 성립한다 — "부분 (max, sum, O) 들을 나중에 합쳐도 한 번에 한 것과 같다". 그래서 KV 를 마음껏 쪼개 병렬 처리하고 마지막에 합치면 된다. FlashAttention 의 분할정복 버전.

10. 요약

Decode (sdpa_vector) 한눈에

  1. 언제: 쿼리 토큰 ≤ 8 (토큰 생성). 보통 qL=1.
  2. 왜 이 커널: 쿼리 1개 → GEMV → memory-bound. MMA 는 낭비. 단순 dot product + simd_sum.
  3. 분할: grid=(B·Hq, 1)=(32,1)=32 TG. 각 TG=1024 thread=32 SG×32 lane. SG 가 K 토큰을(strided), lane 이 head dim 을(4개씩) 나눔.
  4. dot product: lane 이 부분합 → simd_sum 으로 한 키의 score 완성.
  5. online softmax: SG 별로 독립 누적 → 부분 (max, sum, O).
  6. 합치기: SRAM 경유 + transpose 트릭으로 32 SG 결과를 simd_sum 한 방에 합산 후 정규화.
  7. 긴 컨텍스트: 2-pass 로 KV 를 block 분할 병렬 → 합산 (online softmax 결합법칙).
